# Devkit — Contributor Guidelines

## Contributing

1. Fork the repository
2. Create a branch for your skill
3. Submit a PR with clear description of the skill and its use case
4. Each skill should live in `skills/<skill-name>/SKILL.md`

## Skill Structure

```
skills/
  skill-name/
    SKILL.md              # Main reference (required)
    references/           # Supporting files (optional)
```

## General

- One skill per PR
- Test your skill in at least one platform before submitting
- Describe the problem your skill solves, not just what it does
